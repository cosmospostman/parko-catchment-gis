# V10

Joint S1+S2 TAM. Part of a three-way comparison (S2-only / S1-only / joint S1+S2).

**Current best: CVaR25=0.860, macro AUC=0.944** (18 bands incl. s1_vh_vv + s1_rvi, 2026-05-26)

---

## Feature set

18 bands: B02 B03 B04 B05 B07 B08 B8A B11 B12 + NDVI + NDWI + MAVI + NDRE + CI_RE (14 S2, identical to V9) + s1_vh + s1_vv + s1_vh_vv + s1_rvi (4 S1).  
B06 and EVI excluded. Pixel z-score normalisation. Band stats computed per-source (S2 and S1 rows normalised independently).

Sites: Norman Road, Cloncurry (absence only), Etna Creek (train split), Landsend, Lake Mueller, Corfield, Roper River, Hughenden, Burdekin, Maria Downs, Rupert Creek, Frenchs.

## Training history

### Initial — V10 created as joint S1+S2 experiment (2026-05-21)

V10 forked from V9-SPECTRAL at the point where S1 pipeline wiring was complete (mixed-mode `TAMDataset`, per-source DOY normalisation, `is_s1` mask propagated through the Transformer pooling step). V9 frozen as the S2-only comparator.

S1 feature set restricted to raw VH and VV backscatter (`s1_vh`, `s1_vv`) — total 16 input bands. Derived columns (`s1_vh_vv`, `s1_rvi`) excluded until a sweep confirms their value over the raw backscatter pair. Architecture scaled up from V9 (d_model 128→256, n_layers 2→3) to match the increased input dimensionality.

### d_ff sweep — 2026-05-26

`d_ff` was never set in `model_kwargs`, falling through to the `TAMConfig` default of 64 — a carry-over from when `d_model=32`. At d_model=256 this is a 0.25× ratio. Fixed by setting `d_ff=512` explicitly in the experiment. Sweep results (all other params identical, 60 epochs):

| d_ff | CVaR25 | macro AUC | batch_size | notes |
|------|--------|-----------|------------|-------|
| 64   | 0.785  | 0.910     | 8192       | implicit default (baseline) |
| 512  | 0.794  | 0.918     | 8192       | — |
| 1024 | 0.845  | 0.940     | 4096       | **new default**; early stop epoch 53 (best epoch 38) |

d_ff=1024 is a decisive step change (+0.060 CVaR, +0.030 macro over 512). The halved batch size to avoid OOM is not a concern given the result. d_ff=512 was not the ceiling. d_ff=1024 + batch_size=4096 adopted as the experiment default.

### Expanded S1 feature set — 2026-05-26

Added `s1_vh_vv` (cross-pol ratio, dB) and `s1_rvi` (radar vegetation index) to the S1 feature set, bringing the total to 18 bands. Gradient saliency maps confirmed `s1_vh_vv` is the top or near-top S1 feature at multiple sites (Burdekin val, Corfield, Cloncurry); `s1_rvi` contributes at ~half the weight of the raw backscatter channels but is not noise.

| S1 features | CVaR25 | macro AUC | notes |
|-------------|--------|-----------|-------|
| vh, vv (2)  | 0.845  | 0.940     | d_ff=1024 baseline |
| vh, vv, vh_vv, rvi (4) | 0.860 | 0.944 | **current default**; best epoch 43 |

+0.015 CVaR, +0.004 macro. Model ran all 60 epochs (no early stop); best checkpoint at epoch 43.

## Hyperparameter sweep — 2026-05-27

Sweep of dropout and lr against the corrected gate augmentation (additive GateAugDataset, not replacement). All runs at seq=128, batch=2048, p_gate=0.3, T_gate=8.

| run | dropout | lr | CVaR25 | macro AUC | notes |
|-----|---------|----|--------|-----------|-------|
| sweep baseline (gate bug) | 0.5 | 5e-5 | 0.832 | 0.936 | gate was replacing sequences, not augmenting |
| dropout_0.3 + lr_1e4 (partial) | 0.3 | 1e-4 | **0.836** | — | epoch 13/60; OOM-killed; gate bug fixed |

Epoch 3 reached CVaR25=0.824 (~13 min) — significantly faster convergence than any sweep run. Run was killed by system OOM at epoch 15 (8 DataLoader workers × 14 GB dataset). Workers capped at 4 going forward.

## Inference decisions — 2026-05-26

### max_seq_len = 128

Sequence length analysis confirmed that `max_seq_len=128` covers 100% of single-year pixels at Quaids (max=59) and Kowanyama (max=106). The previous V10 checkpoints were trained at `max_seq_len=64`, which was too tight for sites with high S1 cadence: Kowanyama reaches 106 obs/px/yr, and multi-year Mitchell catchment runs (6 years of S2+S1) could reach 150–200. The experiment has been updated to train at `max_seq_len=128`.

The inference code was also padding to `MAX_SEQ_LEN=256` (the module default) regardless of the training cap, wasting 2× compute at T=128. Production inference uses the value from `tam_config.json`.

### Smart S1 subsampling when obs > max_seq_len

When a pixel-year window exceeds `max_seq_len=128` total observations, S2 observations are kept in full and S1 is thinned to fit the remaining budget. S1 subsampling uses greedy farthest-point sampling in DOY space (anchored at the first and last acquisition), preserving seasonal coverage rather than simply truncating the tail. Implemented in `_subsample_s1_indices` in `tam/core/score.py`.

Rationale: S1 has uniform year-round cadence (~30 obs/yr) while S2 is cloud-gated and already sparse. S2 observations represent harder-to-get clear-sky measurements that should never be dropped. Temporally-stratified S1 subsampling preserves the seasonal arc that the model uses for Parkinsonia phenology discrimination. At T=128, single-year pixels are never truncated; multi-year windows (e.g. 6 years of Mitchell data) may require S1 thinning.

### Varlen attention via flash-attn

`TAMClassifier.forward_varlen()` provides a no-padding forward pass using `flash_attn_varlen_func` (flash-attn v4). When `flash_attn` is installed and the device is CUDA, `_gpu_score` automatically packs each mini-batch into flat token tensors (no padding positions) and calls `forward_varlen`. This gives true O(Σ T_i²) attention instead of O(B × T_max²), eliminating the padding overhead entirely.

At `max_seq_len=128` with real Quaids data (p50≈48 obs), varlen removes roughly 60% of padding overhead. It becomes more valuable for Kowanyama-density pixels (p50≈85). The padded `forward()` path remains as fallback when flash-attn is not available (e.g. sm_12x GPUs not yet supported by flash-attn v4 beta).
